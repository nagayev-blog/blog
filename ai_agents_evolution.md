# Эволюция подходов к управлению AI-агентами

## Важная оговорка

Эти подходы **не сменяли друг друга** — они накапливались и наслаивались. Каждый следующий не отменял предыдущий, а добавлял новый уровень абстракции.

---

## 1. Prompt Engineering (2020–2022)

Первый и до сих пор живой подход. Идея: вся логика поведения агента — в тексте промпта.

```
You are a helpful assistant. Think step by step.
Always respond in JSON. Never make up facts.
```

Что умел: zero-shot, few-shot, chain-of-thought, role prompting.

**Почему недостаточно для агентов:** промпт — статичный. Он не реагирует на состояние, не масштабируется, не поддерживает сложную логику ветвления.

---

## 2. RAG / Memory Augmentation (2022–2023)

Попытка решить проблему ограниченного контекстного окна и устаревших знаний.

```python
query → vector search → top-k chunks → inject into prompt → LLM
```

Что дало: агент перестал быть ограничен только весами модели.

**Проблема:** всё ещё реактивный, без состояния между шагами.

---

## 3. Context Management (2023)

Разработчики поняли, что контекстное окно — это дефицитный ресурс, которым нужно управлять явно.

Появились паттерны:
- **Summarization loops** — агент сжимает историю, когда окно заполняется
- **Sliding window** — отбрасывает старые сообщения
- **Selective retention** — сохраняет только "важное"

**Проблема:** кто решает что важно? Логика всё ещё была ad-hoc, без формальной модели памяти.

---

## 4. Tool Use / Function Calling (2023)

Параллельно с context management — может даже важнее. OpenAI добавили function calling, и агенты получили возможность детерминированно вызывать внешние системы.

Это сдвиг парадигмы: агент из **генератора текста** стал **оркестратором действий**.

---

## 5. Agent Frameworks (2023–2024)

LangChain, AutoGen, CrewAI, LlamaIndex — попытка упаковать всё вышеперечисленное в фреймворки. Дали скорость старта, но создали новые проблемы: непрозрачность, сложность отладки, магия под капотом.

---

## 6. Harness Engineering (2024–сейчас)

Реакция на проблемы фреймворков — возврат к явной инфраструктуре, но уже с накопленным опытом. Вместо магии — явные контракты, политики, observability.

---

## Итоговая схема

```
Prompt Engineering                  (2020–2022)
      ↓
  + RAG / Memory Augmentation       (2022–2023)
      ↓
  + Context Management              (2023)
      ↓
  + Tool Use / Function Calling     (2023)
      ↓
  + Agent Frameworks                (2023–2024)
      ↓
  + Harness Engineering             (2024–сейчас)
```
